# 🚀 Day 2 of My DevOps Learning Journey – Understanding SDLC (Software Development Life Cycle)

## 📖 What is SDLC?

The **Software Development Life Cycle (SDLC)** is a structured framework used to design, develop, test, deploy, and maintain software applications efficiently and reliably.

It provides a systematic approach that helps organizations deliver high-quality software while ensuring collaboration between Business, Development, Testing, Operations, and Security teams.

---

## 🎯 Why SDLC Matters

SDLC helps organizations:

✅ Deliver quality software faster
✅ Reduce development risks
✅ Improve project planning and resource utilization
✅ Ensure better collaboration among teams
✅ Maintain software effectively after deployment

---

# 🔄 SDLC Phases

## 1️⃣ Planning Phase

The first stage focuses on understanding business goals and determining project feasibility.

### Activities:

* Requirement Gathering
* Project Planning
* Resource Allocation
* Cost Estimation
* Risk Assessment
* Timeline Planning

### Deliverables:

* Project Plan
* Feasibility Report
* Budget Estimation

---

## 2️⃣ Requirement Analysis & Definition

In this phase, all business and technical requirements are documented in the **Software Requirement Specification (SRS)** document.

### SRS Includes:

* Functional Requirements
* Non-Functional Requirements
* Business Requirements
* Technical Requirements
* Security Requirements

### Outcome:

A clear blueprint of what needs to be developed.

---

## 3️⃣ Design Phase

The system architecture and application design are created before development begins.

### High-Level Design (HLD)

Defines:

* Overall Architecture
* System Components
* Technology Stack
* Database Design
* Integration Flow

### Low-Level Design (LLD)

Defines:

* Detailed Module Logic
* Class Diagrams
* API Specifications
* Database Tables
* Function-Level Implementation

### Example Architecture

```text
User
  ↓
Load Balancer
  ↓
Application Server
  ↓
Database
```

This overall system structure is typically documented in the HLD.

---

## 4️⃣ Development / Build Phase

Developers convert design specifications into working software by writing source code.

### Common Programming Languages

* Java
* Python
* Node.js
* Go
* C#
* JavaScript

### Goal

Transform requirements into a functional application.

---

## 5️⃣ Testing Phase

Quality Assurance (QA) teams validate whether the application meets business and technical requirements.

### Types of Testing

🔹 Functional Testing
🔹 Integration Testing
🔹 System Testing
🔹 Performance Testing
🔹 Security Testing
🔹 User Acceptance Testing (UAT)

### Objective

Identify and fix defects before production deployment.

---

## 6️⃣ Deployment Phase

The application is released to production environments for end users.

### Traditional Deployment

* Manual Releases
* Higher Risk
* Time Consuming

### DevOps Deployment

* Automated CI/CD Pipelines
* Faster Releases
* Reduced Human Error
* Continuous Delivery

### Popular CI/CD Tools

* Jenkins
* GitHub Actions
* GitLab CI/CD
* Azure DevOps

---

## 7️⃣ Maintenance Phase

Once deployed, applications require continuous monitoring and support.

### Activities

* Bug Fixes
* Performance Optimization
* Security Updates
* Feature Enhancements
* Infrastructure Maintenance

### Goal

Ensure long-term application stability and reliability.

---

# 🔁 Change Request (CR) Lifecycle

Software is never truly finished.

When clients request new features or enhancements, the application re-enters the SDLC cycle:

```text
Requirement
      ↓
Design
      ↓
Development
      ↓
Testing
      ↓
Deployment
```

This continuous improvement cycle helps organizations evolve their products over time.

---

# 📊 Popular SDLC Models

### Waterfall Model

A linear and sequential approach where each phase is completed before the next begins.

### Agile Model

An iterative approach focused on continuous delivery and customer feedback.

### Spiral Model

Combines iterative development with risk management.

### V-Model

Extends the Waterfall model by associating each development phase with a corresponding testing phase.

---

# ⚔️ Agile vs Waterfall

| Waterfall                       | Agile                        |
| ------------------------------- | ---------------------------- |
| Sequential Development          | Iterative Development        |
| Fixed Requirements              | Flexible Requirements        |
| Late Testing                    | Continuous Testing           |
| Slow Delivery                   | Faster Delivery              |
| Difficult to Handle Changes     | Easily Adapts to Changes     |
| Customer Involvement is Limited | Continuous Customer Feedback |

---

# 🚀 Agile Methodology

Agile delivers software in smaller iterations called **Sprints**.

Each sprint includes:

* Planning
* Development
* Testing
* Review
* Delivery

### Benefits of Agile

✅ Faster Releases
✅ Better Product Quality
✅ Continuous Customer Feedback
✅ Improved Team Collaboration
✅ Quick Adaptation to Changes

---

# 🏆 Scrum Framework

Scrum is one of the most widely used Agile frameworks.

### Sprint Duration

Typically 2–4 Weeks

### Scrum Roles

👤 Product Owner
👤 Scrum Master
👥 Development Team

---

## Scrum Ceremonies

### Sprint Planning

Define goals and select work for the sprint.

### Daily Standup

15-minute daily progress meeting.

### Sprint Review

Demonstrate completed work to stakeholders.

### Sprint Retrospective

Discuss improvements for the next sprint.

---

# 💡 Key Learning

Understanding SDLC is essential before diving deeper into DevOps because DevOps enhances and automates multiple SDLC phases through:

* Continuous Integration (CI)
* Continuous Delivery (CD)
* Infrastructure as Code (IaC)
* Monitoring & Observability
* Automation

A strong SDLC foundation makes learning DevOps concepts much easier.
